# Migrating to the v2 skill catalog

Version 2 changes the repository from one root-level handbook skill into a
catalog of focused, installable Agent Skills.

## Breaking changes

- The repository root no longer contains `SKILL.md`.
- Installable skills live under `skills/<skill-name>/`.
- Templates live under
  `skills/cuelabs-engineering-standards/assets/templates/`.
- Product readiness moves toward `.cuelabs/project.yaml` rather than being
  inferred from organization-wide prose.

## Migration

1. Remove any manually copied v1 `oss-engineering-standards` skill.
2. Install the v2 catalog:

   ```bash
   gh skill install cuesoftinc/oss-engineering-standards \
     --all --agent codex --scope user --pin v2.0.0
   ```

3. Repeat with `--agent cursor` if Cursor needs a separate user-scope install.
4. Add and review `.cuelabs/project.yaml` in each product repository.
5. Run the new audit tool.
6. Resolve reported baseline gaps without activating planned surfaces.

## Behavior preserved

The existing repository, web, mobile, design, delivery, and QA canons remain
available as focused reference material. The change primarily controls when
that material enters an agent's context and separates product state from
organization policy.

## Rollback

Pin an installation to the latest v1 tag or commit while migrating. Do not mix
v1 and v2 skills in the same client because both may trigger for the same
standardization request.
